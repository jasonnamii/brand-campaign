# D7 — 위기대응 (리콜·사과·재기)

**핵심 목적:** 브랜드 훼손 최소화·신뢰 회복. **72h 프로토콜 + Rebuild 캠페인**

## 특이 전략

| 항목 | 설정 |
|------|------|
| 단계 | Crisis 72h → Stabilize 30일 → Rebuild 6-12개월 |
| 60:40 | Crisis 기간 60:40 재설계 불가 — Holding statement 우선 |
| Rebuild 단계 | L1부터 재착수 (리포지셔닝 D2 연계 가능) |
| 스포크스퍼슨 | CEO·Chief officer 직접 대응이 신뢰 회복 최단 |

## 위기 유형

| 유형 | 예시 |
|------|------|
| 제품 결함 | 리콜·안전 사고 |
| 오너·임원 | 개인 스캔들·갑질 |
| 거짓말·은폐 | 광고 허위·데이터 조작 |
| 사회적 비난 | 차별·부적절 발언 |
| 파트너 위기 | 인플루언서 논란·콜라보 IP 사고 |
| 사이버·개인정보 | 해킹·유출 |

## 72h 프로토콜

| 시간 | 액션 | 산출 |
|------|------|------|
| 0-4h | 사실 확인·피해 범위 파악 | 위기 커뮤니케이션 팀 소집 |
| 4-12h | Holding statement 발표 | "사태 인지·조사 중·추가 발표 예정" |
| 12-24h | 1차 공식 입장·사과 | CEO/Chief officer 공식 발표 |
| 24-48h | 경위·재발방지책 | 구체 행동 (환불·리콜·교체·책임자 조치) |
| 48-72h | 이해관계자 커뮤니케이션 | 고객·직원·투자자·미디어 |
| 72h+ | Rebuild 설계 착수 | L1 재출발 |

## 사과의 7요소 (Kellerman Model)

1. **Acknowledge** — 사실 인정
2. **Apologize** — 진심 사과 ("유감" ≠ 사과)
3. **Empathize** — 피해자 감정 공감
4. **Accept Responsibility** — 책임 수용
5. **Commit to Action** — 구체 행동
6. **Apologize to Affected** — 특정 피해자 직접
7. **Articulate Change** — 재발방지 변화 명시

## 채널 (Crisis 기간)

1. 공식 보도자료·기자회견 (정당성)
2. CEO 직접 영상 메시지
3. 자사 공식 SNS (브랜드 톤 일관)
4. 고객 직접 커뮤니케이션 (이메일·SMS)
5. 광고 일시 중단 (일반 크리에이티브)

## Rebuild 캠페인 (72h 이후)

- L1 재착수 — 새 Purpose·새 POD
- L2 DBA는 일반적으로 유지 (교체 시 "증거 인멸" 인상)
- L3 신뢰 채널 집중 (네이티브·PR·장편 YouTube)
- L4 행동 변화 증거 커뮤니케이션
- L5 Sentiment·NPS·불매율 회복 추적

## 한국 특이

- 오너 리스크 빈발 — 승계·지배구조 문제 (→holdings-consulting)
- 갑질 프레임 — 즉각 해임 표준
- 뒷광고 논란 (2020.9 이후 특히 엄격)
- 불매운동 (NO재팬 등) — 장기 지속

## KPI

| 축 | 지표 | 목표 |
|---|------|------|
| Crisis (72h) | Holding statement 전파 속도 | 24h 내 주요 매체 |
| Stabilize (30일) | Sentiment 회복 | Neutral 이상 50%+ |
| Rebuild (6-12m) | Aided Awareness | 유지 (하락 최소화) |
| Rebuild | NPS 회복 | 위기 전 대비 70%+ |
| Rebuild | 매출 | 위기 전 대비 80%+ |

## Redflag

- 24h 내 Holding statement 없음 → 치명적
- "유감"·"논란"으로 책임 회피 언어
- 스포크스퍼슨 하급 (PR팀장급) → 진정성 실패
- 재발방지 추상적 ("내부 조사 중"만)
- DBA 교체로 증거 인멸 시도 (역효과)
